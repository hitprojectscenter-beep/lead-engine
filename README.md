# מנוע לידים — Lead Engine

מערכת ניהול לידים **אומניצ'אנל** שקולטת פניות מכל ערוץ, מתמללת הקלטות קוליות, מחלצת פרטים אוטומטית, מנקדת, מזהה לקוחות קיימים (Cross-sell/Upsell), ומנהלת את מחזור-החיים של הליד — מקליטה ועד סגירה.

אפליקציה **עצמאית** (Next.js 15 + Drizzle + Postgres + Twilio + OpenAI Whisper). רצה מיד גם ללא שום שירות חיצוני (אחסון JSON מקומי + חילוץ היוריסטי), ומתחברת לשירותים אמיתיים אחד-אחד לפי הצורך.

## הארכיטקטורה — 6 המודולים (האפיון)

### 1. קליטת לידים ואיחוד נתונים (Acquisition)
ערוץ אחד בקוד — `lib/ingest/ingest.ts` — מטפל בכל המקורות דרך אותו צינור:
`תמלול → חילוץ פרטים → זיהוי לקוח קיים → ניקוד → שמירה + ציר-זמן`.

| ערוץ | נתיב | סטטוס |
|------|------|-------|
| ווטסאפ (טקסט) | `POST /api/webhooks/whatsapp` (+ whitelist מספרים אופציונלי) | ✅ עובד |
| ווטסאפ (הקלטה קולית) | אותו webhook — מדיה מסוג audio מתומללת ב-Whisper | ✅ קוד מוכן (צריך `OPENAI_API_KEY`) |
| אפליקציית כיס (PWA) — טופס | מסך `/intake` → `POST /api/intake` | ✅ עובד |
| אפליקציית כיס (PWA) — הקלטה בדפדפן | `/intake` (MediaRecorder) → תמלול Whisper בשרת → ליד | ✅ עובד (תמלול צריך `OPENAI_API_KEY`) |
| דף נחיתה / QR | `POST /api/capture` ← `/l/<slug>` | ✅ עובד |
| מחולל QR | `GET /api/qr?slug=<slug>` → PNG | ✅ עובד |
| הזנה ידנית / הדבקת טקסט חכמה | כפתור "ליד חדש" בלוח | ✅ עובד |
| אימייל / טלפון | `channel` נתמך במודל — webhook להוספה | 🔜 |

### 2. זיהוי לקוחות קיימים (Cross-sell / Upsell)
`store.findCustomerByContact()` משייך פנייה נכנסת לחשבון קיים לפי טלפון/אימייל (מנורמלים ל-E.164). התאמה → תיוג `לקוח קיים`, סיווג **PQL**, ובונוס ניקוד משמעותי. אין כפילויות: פנייה מלקוח/ליד פתוח קיים **ממוזגת** לרשומה אחת.

### 3. העשרה וניקוד אוטומטי (Scoring)
`lib/scoring/score.ts` — מנוע חוקים שקוף, ציון 0–100, עם **הסבר טקסטואלי** לכל ניקוד (מוצג לנציג בכרטיס הליד):
- בסיס לפי ערוץ · שלמות פרטי קשר · מוצר/צורך ספציפי
- **בונוס לקוח קיים (+30)** — האות החזק ביותר, כבקשתך
- איתותי רכישה בטקסט/בתמלול (מחיר, הצעה, דחוף, שדרוג…)
- טמפרטורה (חם/פושר/קר) וסוג (MQL/SQL/PQL) נגזרים מהניקוד וההקשר

### 4. אוטומציה, ניתוב וניהול שלבים (Workflow + Routing)
מחזור-חיים: `חדש → נוצר קשר → מוסמך → בטיפוח → הצעה → נסגר/אבוד`. שינוי סטטוס, שיוך נציג והוספת הערות — מתועדים בציר-זמן. שינוי שדה מהותי → **ניקוד מחדש** אוטומטי.

**ניתוב חכם לנציגים** (`lib/routing/route.ts`) — כל ליד חדש משויך אוטומטית לפי סדר עדיפויות עם נימוק: (1) **המשכיות חשבון** — לקוח קיים נשאר עם מנהל התיק שלו; (2) **התאמת התמחות** — נציג שמומחה למוצר; (3) **אזור**; (4) **איזון עומסים** — הנציג הפנוי ביותר (לא מעמיס מעבר ל-capacity). ניהול נציגים ועומסים בעמוד `/reps`.

### 5. סוכן AI, זמן תגובה וטיפוח (Speed-to-Lead + Nurturing)
כל הודעת ווטסאפ מקבלת **מענה אוטומטי מיידי** (TwiML): אישור קליטה, תמלול, ניקוד וסטטוס. חילוץ הפרטים נעשה ע"י LLM (`gpt-4o-mini`) עם נפילה להיוריסטיקה — כך שליד תמיד נקלט.

**מסעות טיפוח אוטומטיים (Drip)** — `lib/nurture/*` + עמוד `/campaigns`. לידים מצורפים אוטומטית בקליטה למסע לפי בשלות: **warmup** (קר, 3 מסרים), **engage** (פושר, 2), **crosssell** (לקוח קיים, 2). runner שולח שלבים שהגיע זמנם — WhatsApp אמיתי כשיש Twilio, אחרת הדמיה מתועדת. **cron יומי** (`vercel.json` → `/api/cron/nurture`, מוגן ב-`APP_SECRET`/`CRON_SECRET`; Hobby=יומי, Pro=תדיר יותר) + כפתור "הרץ עכשיו". **הפסקה במעורבות**: הודעה חוזרת מליד עוצרת את המסע. חמים/מוסמכים לא מטופחים אוטומטית — עוברים לנציג.

### 6. מדידה ואנליטיקה
`/analytics` — אחוזי המרה, ניקוד ממוצע, פילוח לפי שלב, וטבלת **ROI לפי מקור** (איזה ערוץ מביא את הלידים שנסגרים).

## חיבור שירותים (אופציונלי, אחד-אחד)

**Postgres/Neon** — הגדירו `DATABASE_URL` ואז `npm run db:push`. ללא זה — אחסון JSON ב-`./.data`.

**OpenAI (תמלול קולי + חילוץ חכם)** — הגדירו `OPENAI_API_KEY`. ללא זה — חילוץ היוריסטי, הקלטות לא מתומללות.

**Twilio WhatsApp** — הגדירו `TWILIO_ACCOUNT_SID` + `TWILIO_AUTH_TOKEN`, וכוונו את ה-Sandbox ל-`/api/webhooks/whatsapp` (ראו עמוד "ערוצי קליטה" באפליקציה). ל-production אמיתי צריך מספר WhatsApp Business מאושר (WABA).

## אפליקציית כיס (PWA) — פתיחת ליד מהשטח
מסך `/intake` (מותאם מובייל): טופס מהיר **או הקלטה קולית בדפדפן** (MediaRecorder) שנשלחת ל-`/api/intake`, מתומללת בשרת (Whisper) ויוצרת ליד דרך אותו צינור. ניתן להתקנה כאפליקציה ("הוסף למסך הבית" — `manifest.webmanifest` + `start_url=/intake`). QR להתקנה + סטטוס אינטגרציות בעמוד `/tools`. **אבטחת ווטסאפ:** `WHATSAPP_ALLOWED_SENDERS` (מספרים מופרדים בפסיק) — כשמוגדר, רק מספרים מורשים פותחים לידים; ריק = פתוח לכולם.
